# Контекст
- Требуется Agent Skill для библиотеки Settings by AlexGyver и её кастомного стека зависимостей.
- Нужно сочетание офлайн-готовности (быстрый отклик) и возможности обновления артефактов при выходе новых версий Settings/зависимостей.

# Решение
- Принять гибридный подход:
  - Базовый статичный пакет артефактов (cheatsheet, dist-доки, recipes, templates, playbooks), хранимый в репозитории.
  - Скрипт обновления (`scripts/refresh_gyversettings.(ps1|py)`), подтягивающий свежие исходные доки/README/примеры из GitHub и регенерирующий артефакты.
  - Скилл использует локальные артефакты по умолчанию (офлайн), обновление вызывается вручную/по расписанию.

# Альтернативы
- Чисто статический пакет — быстро, но устаревает (отклонено).
- Онлайновая подгрузка на лету — всегда свежо, но требует сети и может ломаться при изменении структуры репо (отклонено).

# Последствия
- Необходима поддержка скрипта обновления и контроль версий артефактов.
- Агент получает предсказуемый офлайн контекст и может использовать свежие данные после запуска обновления.
