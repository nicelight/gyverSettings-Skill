Мы сможем позже создать Skill  с помощью `$skill-creator`.
Но прежде создания Скилла тебе нужно подготовить артефакты с необходимым контекстом для создания этого Skill. Артефакты будут расположены в [ARTEFACTS.md](spec/docs/ARTEFACTS.md) 
 Название скилла `GyverSettings-skill`
Скилл должен уметь разрабатывать, рефакторить и отлаживать проекты основанные на библиотеке `Settings by Alex Gyver`.  Основная загвоздка в том, что эта библиотека не популярная, имеет около 10 кастомных  зависимостей. 


Библиотека представляет из себя туллсет из: 
- веб сервер (внутри flash esp32) с админ панелью, доступом к памяти контроллера и веб виджетами.
- специфический UI интверфейс в виде подключаемых виджетов
- оптимизированный бекенд
- OTA

Вот ссылка на документацию библиотеки https://github.com/GyverLibs/Settings/blob/main/docs/1.main.md ( это первый файл, есть еще 7 файлов с документацией). Я смогу предоставить всю документацию в одном md файле если понадобится.
Сейчас эта библиотека может быть найдена в реестре как `gyverlibs/Settings`. Библиотека указана в [platformio.ini](platformio.ini) как `gyverlibs/Settings` и установлена со всеми зависимостями в `.pio\libdeps\esp32dev`.  
Библиотека Settings имеет следующие Кастомные зависимости: 
 -    GTL 
 -    GyverDB 
 -    StringUtils 
 -    GyverHTTP 
 -    BSON 
 -    Stamp 
 -    Table 
 -    StreamIO
Все зависимости кастомные, т.е. библиотека Settings это как вершина айсберга, под которым сделан серьезный фундамент из множества эксклюзивных библиотек. 


У нас есть примеры использования библиотеки, но только лишь на их основании сложно понять всю суть работы библиотеки, так как библиотека обширная. 

Я хочу иметь долгосрочную поддержку обновлений, поэтому нужна полная автоматизированная пересборка документации из исходных ссылок на документацию и рабочие примеры + умение перереработать эту документацию и примеры ( возможно, сформировав релевантные рецепты и шаблоны на основании доки + примеров). 
Наш кастомный Agent Skill должен уметь качественно пльзоваться библиотекой Settings, чтобы:
- создавать новый проект 
- рефакторить существующий проект 
- дебажить во время отладки(находить релеватный контекст для белага)
